Understanding the Fork System Call

What is `fork()`?

The fork() system call is a fundamental function in Unix-like operating systems that is used to create a new process. The new process created by fork() is called the child process, and the process that calls fork() is referred to as the parent process.

How Does `fork()` Work?

When a process calls fork(), the operating system performs the following actions:

  1. The operating system creates a new process by duplicating the calling (parent) process.
  2. The child process receives a unique process ID (PID) and a copy of the parent's memory space, including variables, program counter, and file descriptors.
  3. Both processes continue executing from the point where fork() was called.
  4. The return value of fork() is different for the parent and child processes:
    • In the parent process, fork() returns the PID of the child process.
    • In the child process, fork() returns 0.

Example of `fork()`

Here is a simple example of using fork() in a C program:


#include <stdio.h>
#include <unistd.h>

int main() {
    fork(); // Create a new process
    fork(); // Create another new process
    printf("HELLO\n");
    return 0;
}

In this example, the fork() function is called twice, resulting in four processes executing the printf statement, which will print "HELLO" four times.

Key Points

  • fork() is used to create a new process.
  • The child process is a duplicate of the parent process.
  • Both processes continue executing the code after the fork() call.
  • The return value of fork() helps distinguish between the parent and child processes.
